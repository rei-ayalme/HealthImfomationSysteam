# 系统稳定性与数据保障策略 5 分钟演示脚本

## 场景 1：全球宏观视野（交互式全球风险地图 & 预期寿命分布）
- **讲解语**：各位评委好，现在展示的是“交互式全球风险地图”。系统当前采用双层数据源架构。默认优先接入 GEM 与 AIR 真实风险 API。
- **操作**：点击地图，展示正常的响应效果。
- **演示断网/降级**：
  - 后台使用 WireMock 或断开真实 API 模拟 503 错误。
  - **讲解语**：当我们模拟上游接口超时（>5s）或服务不可用时，系统底层的 Data Orchestration Layer（DOL）会瞬间触发熔断。
  - 刷新页面，界面在 200ms 内无感切换至内置的保障数据源（195个国家分级 GeoJSON），图表正常渲染，无白屏。
  - **讲解语**：通过数据质量看板可以看到，降级策略已生效，数据时效性仪表盘实时监控 2021-2023 数据占比。

## 场景 2：中国省级健康指标空间分布
- **讲解语**：这是省级健康指标空间分布。由于现实中统计年鉴数据常有缺失，系统内建了缺失值自动探测机制。
- **操作**：后台修改配置，使得真实数据源中某指标的缺失率 > 20%。
- **讲解语**：当缺失率超过阈值时，系统会自动采用“空间邻接加权平均算法”进行补全，基于“共享省界长度”作为权重。
- **操作**：展示补全前后的对比热力图。
- **讲解语**：补全后，有效数据覆盖率回升至 97% 以上（缺失率 < 3%），满足展示需求。

## 场景 3：微观空间可及性分析（成都市 E2SFCA）
- **讲解语**：下沉到微观层面，在医疗设施 POI 分析中，系统实时抓取高德与百度服务。
- **操作**：截断高德 API（返回 302 或字段不全）。
- **讲解语**：一旦侦测到关键字段缺失或状态异常，模块立即切换至 2023Q4 版内置的 9847 条 POI 数据。
- **操作**：展示 E2SFCA 可达性等值线平滑渲染。
- **讲解语**：底层切换平滑，用户操作（如拖动等时圈）始终保持响应。

## 场景 4：微观群体健康演化仿真
- **讲解语**：最后是群体健康演化仿真模块。
- **操作**：点击“加载仿真环境”。
- **讲解语**：为了应对真实抽样微观库敏感或访问受限的问题，系统运用 Iterative Proportional Fitting (IPF) 算法生成了 100 万规模的合成人口作为替代数据源。初始化在 3 秒内完成，且其年龄、职业与真实普查的相关系数矩阵误差控制在 0.05 以内。
- **操作**：展示界面上的仿真输出曲线。
- **讲解语**：通过双层架构设计，系统实现了 100% 的高可用性与界面无缝衔接。
