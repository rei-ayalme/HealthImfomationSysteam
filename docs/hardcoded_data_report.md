## 主要问题分析

### 1. 数据一致性问题
- **同一国家数据在不同文件中不一致**
  - 中国在macro-analysis中的DALYs率: 3800
  - 中国在meso-analysis中的基础值: [8500, 7200, ...]
  - 数据口径不统一，可能导致用户困惑

### 2. 维护困难
- 所有数据都硬编码在JavaScript中
- 修改数据需要修改代码并重新部署
- 无法通过配置或API动态更新

### 3. 可扩展性差
- 国家列表固定，添加新国家需要修改多处代码
- 指标配置结构固化，新增指标成本高

### 4. 数据准确性风险
- 硬编码数据可能是演示数据，与真实数据有偏差
- 数据更新不及时，可能过期

---

## 改进建议

### 短期方案
1. **提取配置对象**: 将硬编码的国家列表、指标配置提取为独立的CONFIG对象
2. **统一数据来源**: 确保同一指标在不同文件中使用相同的数据
3. **添加注释**: 标注哪些数据是演示数据，哪些是真实数据

### 长期方案
1. **数据API化**: 将所有硬编码数据迁移到后端API
2. **配置中心**: 建立前端配置中心，支持动态配置
3. **数据管理后台**: 提供数据管理界面，支持非技术人员更新数据
4. **数据验证**: 建立数据验证机制，确保数据准确性
