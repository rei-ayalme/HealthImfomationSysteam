## 四、数据质量验证

### 4.1 卫生年鉴表数据失真问题

#### 4.1.1 问题描述

⚠️ **严重数据缺陷**: 系统存在 `data/raw/卫生年鉴表/2012.xlsx` 等多个年份的卫生年鉴Excel文件，但代码中**并未实际读取这些文件**。

#### 4.1.2 实际数据流向

| 预期数据流 | 实际数据流 | 状态 |
|-----------|-----------|------|
| 卫生年鉴Excel → 数据提取 → 数据库 → API | 硬编码Python字典 → 直接API返回 | ⚠️ **数据缺失** |

#### 4.1.3 代码证据

**settings.py 中配置了年鉴路径但未使用**:
```python
# settings.py#L63
YEARBOOK_DATA_PATH = os.path.join(RAW_DATA_PATH, "卫生年鉴表")

# settings.py#L65-72
STANDARD_COLUMN_MAPPING = {
    "physicians": ["执业(助理)医师", "执业医师", "医师"],
    "nurses": ["注册护士", "护士"],
    "hospital_beds": ["床位数", "医疗卫生机构床位数"],
    "population": ["人口数", "年末人口数", "总人口"],
}
```

**但实际代码中无任何读取卫生年鉴文件的逻辑**，所有国家数据均来自硬编码字典。

### 4.2 数据单位核查

#### 4.2.1 单位一致性检查

| 字段 | 标注单位 | 实际值 | 单位一致性 | 核查结果 |
|------|---------|--------|-----------|---------|
| physician | 人/千人口 | 2.99 | ✅ 一致 | 正确 |
| bed | 张/千人口 | 7.1 | ✅ 一致 | 正确 |
| population | 百万 | 1410 | ✅ 一致 | 正确 |
| expectancy | 岁 | 79.0 | ✅ 一致 | 正确 |
| ncd_ratio | % | 89.4 | ✅ 一致 | 正确 |

#### 4.2.2 数据库模型单位定义

```python
# db/models.py#L20-23
physicians_per_1000 = Column(Float, comment="每千人口执业(助理)医师数")
nurses_per_1000 = Column(Float, comment="每千人口注册护士数")
hospital_beds_per_1000 = Column(Float, comment="每千人口医疗卫生机构床位数")
population = Column(Float, comment="总人口(万人)")
```

⚠️ **注意**: 数据库模型定义的是 `population(万人)`，但 `meso.py#L105` 中硬编码数据标注为 `百万`，存在**单位不一致**问题。

| 来源 | 人口单位 | 中国人口值 |
|------|---------|-----------|
| db/models.py | 万人 | 141000 |
| meso.py 硬编码 | 百万 | 1410 |
